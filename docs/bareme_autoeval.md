# Barème d'évaluation de l'archive de rendu de projet

## Décomposition (4pts)
qualité de décomposition en fonctions et types, réutilisation du code à bon escient, paramétrabilité
> 4pts 

## Volume de Travail (6pts)
appréciation du volume de travail et du temps passé 
> 6pts
> Yann : 30h (types customs et sauvegarde + chargement de sauvegarde)
> Anaelle : 27h (interface + ctrl de saisie)

## Externalisation et persistance des données (3pts)
usage de fichiers dans le projet, sauvegarde
> 3pts
> Bien organisée et claire

## Clarté du code (2pts)
pertinence des noms de variables, constantes et fonctions
> 2pts
> oui.

## Tests (2pts)
présence et pertinence des tests
> 0pts
> y'en a pas encore mqis y'aura

## Structure de l'archive (1pt)
Répertoires, scripts, capture écran
> 0.5pts 
> y'aura un dossier shot mais pour l'instant c'est pas fini, y'a un run par plateforme pour lancer le jeu

## README (1pt) 
qualité du README
> 1pts
> suffisant pour une alpha mais ça demande plus d'explication de code (exemple, descriptions des types, un peu de docs quoi...)

## Contrôle de saisie (1pt) 
contrôle de saisie (ie. pas d'exception)
> 1pts
> Oui, pas d'erreur pour l'instant
